# .gitignore

**Official docs:** [gitignore — Specifies intentionally untracked files to ignore](https://git-scm.com/docs/gitignore)

## What it does

`.gitignore` is a file at the root of the repo that tells Git which files and folders to **never track**. Files listed here are invisible to Git — they will never appear in `git status`, never get staged, and never get committed.

---

## Common files to ignore

```
node_modules/       # dependencies — never commit these (huge, always regenerated by npm install)
dist/               # build output — generated by the build command
.env                # environment variables and API keys — security risk if pushed
.env.local          # local environment overrides
.DS_Store           # macOS system file
Thumbs.db           # Windows system file
*.log               # log files
.angular/           # Angular CLI cache
```

### Why never commit node_modules?

It can contain thousands of files and hundreds of megabytes. Anyone who clones your project runs `npm install` to regenerate it from `package.json`. Pushing it would make every clone enormous and slow.

---

## Pattern syntax

```
# Comment line
node_modules/       # ignore the folder and everything inside it
*.log               # ignore all files ending in .log
!important.log      # but NOT this specific log file (negation)
/dist               # ignore dist/ only at the root (not in subfolders)
src/**/*.test.js    # ignore all .test.js files inside src/
```

---

## When to create it

**Before your first commit.** If you create `.gitignore` after already committing a file, adding it to `.gitignore` does not remove it from the history — Git keeps tracking it.

If you committed a file by mistake:

```bash
git rm --cached filename     # stop tracking the file (removes from Git, keeps on disk)
# then add filename to .gitignore
git commit -m "chore: stop tracking .env file"
```

`--cached` means "remove from Git's index only" — the file stays on your disk, it just becomes untracked from now on.

---

## GitHub's gitignore templates

When creating a new repo on GitHub, you can choose a `.gitignore` template for your stack (Node, Angular, Java, etc.). It pre-fills the most common patterns for that technology. Always start with the template and add your own entries on top.
